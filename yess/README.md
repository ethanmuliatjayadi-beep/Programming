# Year 1: The Integrated Foundation

# Months 1–3: Automation, Environments, and Logic

Cyber Focus: Install a Linux virtual machine (e.g., Ubuntu) and master the command line (Bash). You must become comfortable navigating an OS without a mouse.

AI Focus: Level up your Python. Learn Object-Oriented Programming (OOP) and error handling. These are essential for building scalable AI models and security scripts later.

Monthly Integration Project: Write a Python script that runs in your Linux environment to automatically scan a folder, identify different file types, and log their metadata into a structured file.

# Months 4–7: Networking and Data Manipulation

Cyber Focus: Learn how the internet works—IP addresses, DNS, HTTP/HTTPS, and the OSI Model. Understand ports and protocols like TCP vs. UDP.

AI Focus: Start using Python libraries like Pandas for data manipulation and Matplotlib for graphing. AI is built on data, so you need to know how to clean and visualize it.

Monthly Integration Project: Use Python to write a basic port scanner. Take the results of that scanner and use Pandas to create a chart showing which ports are most commonly open on your network.

# Months 8–12: Security Principles and Algorithmic Thinking

Cyber Focus: Study the CompTIA Security+ curriculum. Focus on the CIA Triad (Confidentiality, Integrity, Availability), cryptography basics, and identifying threat actors.

AI Focus: Introduce yourself to Scikit-learn. Learn the logic behind basic algorithms like Linear Regression and Decision Trees.

Monthly Integration Project: Build a Python tool that hashes text for security (cryptography) and then use a simple Decision Tree to "classify" whether a password is weak or strong based on its length and character variety.

# Strategy for Consistency

The 30-Minute Rule: Commit to at least 30 minutes of focused learning every single day. This prevents burnout while keeping the concepts fresh.

The 2:1 Build Ratio: For every hour you spend watching a tutorial, spend two hours actually writing code or working in your Linux terminal.

Document Your Progress: Use a GitHub repository to save every script you write. Explaining what your code does in a simple README file is a professional skill that will pay off in Year 4.
